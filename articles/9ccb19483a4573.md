# 開発の経緯
自分のコーディング能力を上げようと思い、ブログを作成することにしました。
しかし、AIを使っていくうちに「1日でどこまでサービスができるのか気になる..」と思い始めてしまいました。
気が付けば、次の日にはブログサービスが完成してしまったので、本記事を書いています。


# 開発の流れ
メインは「Manus」のAIを使用して作成しました。
事前にブログの基礎的な仕組みをGPT5と共同で作成しました。
GitHubのリポジトリにあるマークダウンの記事を表示する機能をコア機能として開発し、ある程度できたらGitHubにpushしておきました。
その後、そのリポジトリをManusに渡して10時間くらい格闘しました。

# ブログのコンセプト
ユーザーファーストなブログを作ることを目指しました。
ブログを書くユーザーはGitHub上で記事を管理することでブログサービスに依存せずに自分の記事を持ち続けることができます。
また、ブログを読むユーザーは自分の好きなレイアウトで読めるようにしたいと考えました。
これらは基本的な方針となり、ブログを作成できました。


# 苦労したこと
かなり会話をしながら作成したため、コンテキスト量が膨大になってしまいました。
そのため、途中でManusがソースコードの保存をやめてしまい、混乱してしまう場面がありました。
gitでバージョン管理しているので、細かく保存させることをお勧めします。


# よかったこと
優秀なAIのおかげでコンセプト通りのブログが作成できました。
会話ベースでここまで作りこめるとは思っていいなかったのでとても良い発見でした。
